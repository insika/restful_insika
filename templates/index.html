RESTful INSIKA Interface Logging

NOTE: This server is designed for development and conformance tests on 
application level only. Do not use in productive environments!

Log messages can be accessed via: 
http://[server]:[port]/[messageKey] 
while 'messageKey' is the server's unix time.

Lists of shifts and trips can be accessed via: 
http://[server]:[port]/insika/[tpId]/[tpIdNo]/shifts/
http://[server]:[port]/insika/[tpId]/[tpIdNo]/trips/ 

Message bodies (INSIKA XML messages) can be accessed via: 
http://[server]:[port]/insika/[tpId]/[tpIdNo]/shifts/[seqNoReport] 
http://[server]:[port]/insika/[tpId]/[tpIdNo]/trips/[seqNoTransaction] 
or by following the corresponding links on this page.

The serverlist can be accessed via: 
http://[server]:[port]/insika/servers 
http://[server]:[port]/insika/[tpId]/[tpIdNo]/servers/
Registered tpId/tpIdNo's are:
$for user in userList: $user
